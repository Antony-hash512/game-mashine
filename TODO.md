# План реализации MVP Платформера (Godot 4)

Этот план содержит пошаговый список задач для создания простейшего платформера с использованием обновленных ассетов (размер ячейки `180x180` пикселей).

## 📋 Список задач (TODO)

### 1. Настройка проекта и управления
- [x] **Настройка управления (Input Map)**: Добавить в настройки проекта `project.godot` клавиши для действий:
  - `move_left` (клавиши A, Left Arrow)
  - `move_right` (клавиши D, Right Arrow)
  - `jump` (клавиши Space, W, Up Arrow)
- [x] **Настройка экрана**: Включить сглаживание пикселей (Nearest) для сохранения четкости пиксель-арта и установить режим растягивания `canvas_items` (уже частично настроено).

### 2. Создание игрока (`player.tscn` / `player.gd`)
- [x] **Сцена игрока**: Создать `CharacterBody2D` с дочерними нодами:
  - `Sprite2D` (текстура `player.png`, Hframes = 9, Vframes = 7)
  - `CollisionShape2D` (коллайдер по форме тела)
  - `Camera2D` (следует за игроком с небольшим сглаживанием)
- [x] **Скрипт игрока**:
  - Реализовать базовую физику: гравитация, горизонтальное движение, ускорение/замедление, прыжок.
  - Добавить смену кадров (`frame` или `frame_coords`) для анимаций:
    - **Idle**: циклическое проигрывание (например, кадры 0–3)
    - **Run**: циклическое проигрывание (например, кадры 8–15)
    - **Jump/Fall**: фиксированные кадры в воздухе
  - Реализовать поворот персонажа (`flip_h` у спрайта) в зависимости от направления движения.
  - Добавить проигрывание звука прыжка (`sfx_jump_16bit_quality_01.wav`).
- [x] создание тестового уровня для ручного тестирования и отладки.
  - Придумать формат хранения уровня например в xml, json, yaml или toml. С такой разметкой, чтобы данный формат был удобен для редактирования человеков. Или вместо придумывания своего формата использовать встроенные инструменты godot, если они подходят для данной задачи.
  - Создать тестовый уровень состоящего только из платформ и игрока. Без монет, врагов и ям (его можно сделать склеенным слева и справа для обеспечения возможности движения в обе стороны).
  - Сделать так, чтобы данный тестовый уровень автоматически загружался и запускался сразу после запуска игры.

Примечание: к последующим шагам 3-6 пока что не приступаем потому что надо сначала реализовать несколько других доработок.

### 3. Создание интерактивных элементов (Монеты)
- [ ] **Сцена монеты (`coin.tscn` / `coin.gd`)**:
  - Создать `Area2D` -> `Sprite2D` (`interactables.png`, Hframes=8, Vframes=6, кадр монетки) -> `CollisionShape2D`.
  - Добавить скрипт: при входе игрока (`body_entered`) проигрывать звук получения монетки (`sfx_coin_16bit_quality_01.wav`), отправлять сигнал в HUD/менеджер уровня и удалять монетку (`queue_free()`).

### 4. Создание врагов (Паук)
- [ ] **Сцена врага (`enemy_spider.tscn` / `enemy_spider.gd`)**:
  - Создать `CharacterBody2D` (или `Area2D`) -> `Sprite2D` (`enemy_spider.png`, Hframes=8, Vframes=6) -> `CollisionShape2D`.
  - Добавить `RayCast2D` перед врагом для обнаружения краев платформы и стен.
- [ ] **Поведение врага**:
  - Патрулирование: движение влево-вправо, разворот при встрече со стеной или при приближении к краю платформы.
  - **Механика победы**: Если игрок прыгает на врага сверху — враг погибает (звук `sfx_enemy_stomp_16bit_quality_01.wav`, игрок отскакивает вверх, враг удаляется).
  - **Механика поражения**: Если игрок сталкивается сбоку/снизу — игрок получает урон / уровень перезапускается.

### 6. Интерфейс (HUD) и победа/поражение
- [ ] **HUD (`hud.tscn`)**: Простейший интерфейс для отображения собранных монет.
