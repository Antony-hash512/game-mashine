# Документация по формату sprites.xml

Файл `sprites.xml` используется для описания структуры спрайтов, анимаций и их кадров. На основе этого XML-файла скрипт `convert_sprites.py` генерирует готовый `sprites.json`, который используется в игре на Godot.

---

## Общая структура XML

```xml
<?xml version="1.0" encoding="UTF-8"?>
<sprites>
  <!-- Группа объектов (например, игрок, монета, враг) -->
  <group object="имя_объекта">
    
    <!-- Набор кадров для конкретного действия / анимации -->
    <set action="имя_действия">
      <file>имя_файла_текстуры.png</file>
      
      <!-- Частота кадров (выберите что-то ОДНО: либо fps, либо spf) -->
      <fps>12</fps> <!-- Кадров в секунду -->
      <!-- <spf>0.08</spf> --> <!-- Секунд на один кадр -->
      
      <!-- Описание кадров -->
      <frame>
        <from tile="R5:C0" />
        <extend top="1" />
      </frame>
      
      <!-- Повторяющиеся кадры с использованием относительных смещений -->
      <frame>
        <from tile="same:next" repeat="7" />
        <extend top="1" repeat="7" />
      </frame>
      
    </set>
  </group>
</sprites>
```

---

## Теги и их атрибуты

### 1. `<sprites>` (Корневой тег)
Корневой контейнер для всех объектов. Должен присутствовать в единственном экземпляре.

### 2. `<group>` (Группа спрайтов)
Определяет логический игровой объект.
* **Атрибуты**:
  * `object` (обязательный) — уникальное имя объекта (например, `player`, `coin`).

### 3. `<set>` (Анимация / Действие)
Группирует кадры для конкретного действия объекта.
* **Атрибуты**:
  * `action` (обязательный) — имя действия (например, `idle`, `running`, `jumping`).
* **Внутренние теги**:
  * `<file>` (обязательный) — имя графического файла (например, `player_alt.png`). Файл должен находиться в папке `assets/images/`.
  * `<fps>` (опциональный) — скорость воспроизведения в кадрах в секунду.
  * `<spf>` (опциональный) — длительность одного кадра в секундах.
  > [!IMPORTANT]
  > Нельзя указывать `<fps>` и `<spf>` одновременно в рамках одного `<set>`.

### 4. `<frame>` (Кадр или группа кадров)
Описывает границы одного или нескольких кадров на текстуре.
* **Внутренние теги**:
  * `<from>` (обязательный) — базовая ячейка спрайта.
  * `<extend>` (опциональный) — смещение границ кадра в ячейках.

---

## Координаты и позиционирование (`<from>` и `<extend>`)

Каждый кадр привязан к сетке ячеек. Размеры одной ячейки по умолчанию составляют **180x180 пикселей** (задается константами `CELL_W` и `CELL_H` в конвертере).

### Тег `<from>`
Задает базовую плитку (tile), от которой ведется расчет.
* **Атрибуты**:
  * `tile` (обязательный) — координата ячейки. Поддерживает абсолютный и относительный форматы:
    * **Абсолютный формат**: `R<номер_строки>:C<номер_колонки>` (например, `R5:C0` — 5 строка, 0 колонка). Нумерация начинается с `0`.
    * **Относительный формат**: использует ключевые слова `same` (оставить без изменений), `next` (предыдущее значение + 1) и `prev` (предыдущее значение - 1). 
      * Пример: `same:next` — та же строка, следующая колонка относительно предыдущего обработанного кадра.
      * Пример: `Rsame:Cnext` — альтернативная форма записи с префиксами.
  * `repeat` (опциональный) — указывает, сколько раз повторить этот кадр со смещением. Используется совместно с относительными координатами.

### Тег `<extend>`
Позволяет расширить рамку кадра за пределы базовой ячейки. Это полезно, если персонаж или объект в данной анимации выходит за стандартные границы 180x180 (например, высоко поднимает руки или имеет длинный хвост).
* **Атрибуты**:
  * `top`, `bottom`, `left`, `right` (опциональные) — на сколько ячеек расширить рамку в соответствующую сторону. Значение должно быть целым положительным числом.
  * `repeat` (опциональный) — должно соответствовать значению `repeat` в теге `<from>` для корректного применения ко всем генерируемым кадрам.

---

## Пример оптимизации с `repeat`

Вместо длинного ручного перечисления кадров:

```xml
<!-- Длинная запись -->
<frame><from tile="R5:C0" /><extend top="1" /></frame>
<frame><from tile="R5:C1" /><extend top="1" /></frame>
<frame><from tile="R5:C2" /><extend top="1" /></frame>
<frame><from tile="R5:C3" /><extend top="1" /></frame>
<frame><from tile="R5:C4" /><extend top="1" /></frame>
<frame><from tile="R5:C5" /><extend top="1" /></frame>
<frame><from tile="R5:C6" /><extend top="1" /></frame>
<frame><from tile="R5:C7" /><extend top="1" /></frame>
```

Можно использовать лаконичную запись с `repeat="7"` и относительным шагом `same:next`:

```xml
<!-- Оптимизированная запись -->
<frame>
  <from tile="R5:C0" />
  <extend top="1" />
</frame>
<frame>
  <from tile="same:next" repeat="7" />
  <extend top="1" repeat="7" />
</frame>
```
